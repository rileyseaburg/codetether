## Installation Methods

<Tabs>
  <Tab title="Cargo (Recommended)">
    Install from crates.io:
    ```bash
    cargo install codetether-agent
    ```
    
    The binary will be installed to `~/.cargo/bin/codetether`.
  </Tab>
  
  <Tab title="From Source">
    Build from source for the latest features:
    ```bash
    git clone https://github.com/codetether/codetether-agent
    cd codetether-agent
    cargo build --release
    
    # Binary at ./target/release/codetether
    # Optionally copy to PATH:
    sudo cp target/release/codetether /usr/local/bin/
    ```
  </Tab>
  
  <Tab title="Docker">
    Run in a container:
    ```bash
    docker run -it \
      -e VAULT_ADDR="$VAULT_ADDR" \
      -e VAULT_TOKEN="$VAULT_TOKEN" \
      -v $(pwd):/workspace \
      ghcr.io/codetether/agent:latest \
      --server https://api.codetether.run
    ```
  </Tab>
  
  <Tab title="Nix">
    Using Nix flakes:
    ```bash
    nix run github:codetether/codetether-agent
    ```
  </Tab>
</Tabs>

## Verify Installation

```bash
codetether --version
# codetether 0.1.0
```

## Configuration

Configuration is stored in `~/.config/codetether-agent/config.toml`:

```toml
# Default AI provider
[default]
provider = "anthropic"
model = "claude-sonnet-4-20250514"

# A2A settings
[a2a]
enabled = true
auto_connect = true

# UI preferences
[ui]
theme = "dark"

# Session settings
[session]
auto_save = true
history_limit = 100
```

<Warning>
  **API keys are NOT stored in config files.**
  
  All provider API keys must be stored in HashiCorp Vault. See [Vault Setup](/agent/vault-setup) for details.
</Warning>

### Initialize Configuration

Create default configuration:

```bash
codetether config --init
```

### View Configuration

```bash
codetether config --show
```

## Command Reference

### Global Options

| Option | Description |
|--------|-------------|
| `--server <URL>` | A2A server URL |
| `--token <TOKEN>` | Authentication token |
| `--name <NAME>` | Worker name |
| `--print-logs` | Print logs to stderr |
| `--log-level <LEVEL>` | Log level (DEBUG, INFO, WARN, ERROR) |

### Commands

| Command | Description |
|---------|-------------|
| (default) | A2A worker mode |
| `tui` | Interactive terminal UI |
| `serve` | Start HTTP API server |
| `run` | Non-interactive mode |
| `worker` | Explicit A2A worker mode |
| `config` | Configuration management |

### Examples

```bash
# Connect to platform with authentication
codetether --server https://api.codetether.run --token $TOKEN

# Run with debug logging
codetether --server https://api.codetether.run --log-level DEBUG --print-logs

# Start TUI in a specific project
codetether tui /path/to/project

# Run a task non-interactively
codetether run "fix the failing tests"

# Start API server on custom port
codetether serve --port 8080 --hostname 0.0.0.0
```

## Environment Variables

| Variable | Description |
|----------|-------------|
| `CODETETHER_SERVER` | A2A server URL |
| `CODETETHER_TOKEN` | Authentication token |
| `CODETETHER_WORKER_NAME` | Worker name |
| `VAULT_ADDR` | HashiCorp Vault address |
| `VAULT_TOKEN` | Vault authentication token |
| `VAULT_MOUNT` | KV secrets mount (default: `secret`) |
| `VAULT_SECRETS_PATH` | Secrets path prefix (default: `codetether/providers`) |
